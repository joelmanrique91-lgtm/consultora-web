# Guía de contribución

Reglas simples para mantener el repositorio ordenado y consistente.

## Commits
- Mensajes claros y cortos, en inglés.
- Usar verbos en presente: Add, Update, Fix, Remove.
- Un commit por cambio lógico.

## Estructura del proyecto
- Documentación general en `docs/`
- Código del sitio web en `site/`

## Flujo de trabajo (fase inicial)
- Cambios directos sobre `main`.
- Sin ramas ni pull requests mientras el proyecto esté en definición.

Este flujo se revisará cuando el proyecto crezca.
